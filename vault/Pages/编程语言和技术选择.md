前端无非是 HTML、CSS、JavaScript, 最终目标为独立 Developer.

> JavaScript 糟粕太多了

后端则通常以数据库、中间件为主, 还需要懂操作系统、网络等知识, 最终目标为 Architect.

显然后端的复杂度和要求比前端更高.

> developer 常有, 而 architect 不常有.

前端技术相对而言更为简单纯粹, 但又较为繁杂需要扣细节.

前端更容易取得成果, 也更不容易被技术异化影响. 对后端则可以徐徐图之.

前端的主要应用方向
- 网站
- app
- 小程序
- 插件/主题

后端语言选择
- C++ 过于原始且复杂
- Rust 偏底层
- Go 不喜
- Java/Scala
- Python 本质上还是脚本语言, 并发性能很弱

> [!note] 
> - 前端 js/ts
> - 后端 java、go
> - 底层 c/cpp、rust
> - 脚本 python、lua

综合来看, 选 rust.

but, 问题还是做什么, 或者说想参与到什么中去.
