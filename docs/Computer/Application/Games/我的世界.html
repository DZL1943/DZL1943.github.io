我的世界

安装​

我的世界分为 Java 版和基岩版两个版本, 可以简单理解为电脑版和手机版.

Java 版(JE)​

  • 1.21(下个版本)
  • 1.20.1(当前推荐版本)
  • 1.19
  • 1.17
  • 1.12

Java 版顾名思义首先得安装 Java(JDK), 这边推荐 Temurin

然后安装启动器(用来下载和管理游戏版本、模组等), 推荐 HMCL(支持 m1)、PrismLauncher、FCL(适用于安卓移动端)、PojavLauncher(适用于安卓、iOS)

基岩版(BE/PE)​

  • 1.20.51(当前推荐版本)

包括网易版和国际版, 推荐后者, 可在苦力怕论坛下载.

[!important]+ 建议修改存储为外部(否则在 Android/data 中无法查看存档)
存档导出模组: https://llama-studios.github.io/mobile-export/

Toolbox ζ 汉化版(支持1.20.51)版本更新！ - 软件资源 - Minecraft(我的世界)苦力怕论坛 (klpbbs.com): 一个基岩版外挂, 可惜闪退用不了

根据运行方式的不同, 有的模组仅需要客户端安装, 有的模组需要在服务器安装.

光影​

光影基于渲染模组, 主流的渲染模组为

  • OptiFine(常用于 Forge)
  • Iris(推荐, 常用于 Fabric)

推荐一个低配光影: MakeUp(支持 Iris, 比 BSL、Photon(直接下载 zip 文件即可) 帧率更好). 正常配置直接上 Complementary.

如遇模组效果不显示, 建议重载光影.

基岩版参考这个(建议放弃) https://mcpedl.com/newb-shader/

红石电路​

红石元件​

  • 电源: 本身相当于强充能方块, 并且能够给其他方块强充能; 可以改变红石线指向
    • 红石块(非实体): 性质和强充能方块类似，与红石块相邻的红石导体_不会_被充能
    • 红石火把: 红石火把能够强充能其_上方_的红石导体，激活除附着以外的毗邻元件。
      • 红石火把有2游戏刻的延迟
      • 在Java版中，若是红石火把亮灭交替过快，红石火把会“烧毁”
      • 两个红石火把可以模拟中继器
      • 火把塔
    • 侦测器(非实体)
    • 探测器
    • 拉杆: 当开启时，拉杆会强充能其附着的方块，激活毗邻的朝向正确（如果需要）的元件。
    • 按钮
      • 石质按钮一旦被按下，会保持其状态达10刻（1秒）；木质则为15刻（1.5秒），但如果是被发射的箭或三叉戟激活的话，木质按钮会保持激活，直到箭或三叉戟被捡起或在1分钟之后自然消失。
    • 压力板
      • 石质压力板只会被生物（包括玩家）激活，而木质和测重压力板还会被掉落的物品、发射的箭或钓鱼竿的鱼钩等实体激活。
    • 绊线钩
    • 陷阱箱
    • 唱片机: 不同的唱片对应不同的能量
    • 讲台
    • 标靶
    • 避雷针
  • 传输元件
    • 红石粉: 激活的红石线会激活指向的机械元件，并对指向的红石导体“弱充能”至自身信号强度
    • 中继器: 红石中继器用来传输信号，重新加强因距离而减弱的红石信号，延迟信号，和给予红石信号单向的传输方向。
      • 中继器只对背面的方块作出反应，对前面的方块传输信号（箭头的方向）。它的延迟是可调节的，通过右键它可将延迟设置成1至4刻（默认为1刻）。
      • 一个充能的红石中继器激活在它前面的红石粉和机械，或者强充能一个红石导体。它对在它下面，上面，侧面，背面的方块无影响。
      • 红石中继器不仅“中继”一个信号（为了传输重新加强），它也会产生1至4刻的延迟。红石中继器也会把任何持续时间短于它的延迟的脉冲增长到与它的延迟相符合。
      • 红石中继器能通过在它的侧面用另一个红石中继器或红石比较器“锁存”当前状态。一个锁定的中继器会保持状态直到解锁。
    • 比较器: 四大用途：保持信号强度(不同于中继)、比较信号强度、信号强度作差以及检测方块的状态
      • 红石比较器共有三种端口：后侧的主输入端 I，两侧的副输入端 C，前侧的输出端 O。红石比较器顶部纹理的箭头由主输入端指向输出端。
      • 红石比较器有两种模式。点击比较器可以在比较模式（前端红石火把熄灭）和减法模式（前端红石火把点亮）之间切换。
        • 比较: 选择信号最大的与主输入端比较, 主输入端小则不予输出
        • 减法: 选择信号最大的与主输入端作差并输出(大于主输入端则为负不输出)
      • 发出信号的比较器会激活前方的红石粉、机械元件、背对的红石中继器或比较器，也可以强充能(有充能等级)前方的红石导体。对其他位置的方块没有影响。
  • 机械元件
    • 红石灯(实体)
    • 发射器(实体)
    • 投掷器(实体)
    • 活塞(非实体)
    • 漏斗(非实体)
    • 音符盒(实体)
    • 命令方块(实体)
    • TNT
    • 铁轨
    • 激活铁轨、动力铁轨

充能与激活​

当一个方块被充能时，这个方块毗邻的所有机械元件都可以被激活，被充能的方块叫做充能方块。

能够被充能的方块就被称为红石导体（也叫做实体方块）。大多数红石导体都是不透明的方块。一些透明方块可布线(但不会被充能), 其中一些(半砖/台阶、漏斗、萤石)在垂直方向只能向上传输.

  • 强充能的方块可以激活毗邻的机械元件和红石元器件。红石导体可以被电源及中继器、比较器强充能。被强充能的方块所表现性质和红石块大体相同，但是红石块不是红石导体，且可以改变红石线的形状和指向、向红石比较器边侧输入信号[仅 Java 版]，而强充能方块不能。
  • 当红石导体仅被红石线充能，我们称这个方块被弱充能。弱充能与强充能的区别在于，==弱充能方块无法激活毗邻的红石线==。

充能方块无法再去充能其他方块，只有电源和传输元件才可以去充能一个方块。

使用多少信号强度的信号去充能一个方块，我们就说这个方块有多少充能等级。被强充能的方块有多少充能等级，就可以激活毗邻的红石粉至多少信号强度。充能等级的大小与充能的强与弱完全无关。

红石导体在以下的任意一种情况中，都是被强充能了：

  • 开启的电源的附着（如果有）。
  • 输出信号的红石火把、红石比较器、红石中继器的指向。
  • 开启的陷阱箱和正在翻页的讲台下方的方块。
  • 发出红石信号的侦测器背朝的方块。
  • 红石块本身（属于红石绝缘体）。

红石导体在以下的任意一种情况中，都是被弱充能了：

  • 红石线的附着。
  • 红石线的指向。

每个充能方块都可以传输信号到某些毗邻方块。但传递过程可能因方块本身不同而出现以下特定规则：

  • 除测重压力板、陷阱箱之外的电源和红石中继器只能使红石导体强充能至15等级。
  • 红石线是多少信号强度，被其充能的方块就被弱充能至多少充能等级。
  • 测重压力板、陷阱箱和红石比较器输出的信号是多少强度，被充能的方块就被强充能至多少充能等级。

红石导体除了能被充能这一种性质外，还有使在激活的动力铁轨上的矿车加速离开另一端的红石导体[仅Java版]、允许红石比较器检测后方方块、“压断”红石线、阻止箱子开启的性质。红石线不能穿过红石导体与斜上方的红石线连接，箱子和末影箱不能在红石导体下方开启。

一个机械元件可以被其毗邻激活
激活该机械元件的充能方块的充能等级的大小并不会影响激活结果

对于属于红石导体的机械元件（包括命令方块、投掷器、发射器、音符盒与红石灯等等），既可以被充能也可以被激活。因此区分它们是被激活还是被充能相当重要：

  • 如果它能够激活毗邻的红石元器件和机械元件，那么它就被充能了。
  • 如果它本身作出了一定的反应，那么它就被激活了。

机械元件只要被充能就一定也会被激活，但被激活并不会一定会被充能（如被毗邻的充能方块激活）。

不是红石导体的机械元件（门、栅栏门、活塞、漏斗、铁轨、活板门等等）可被激活并作出反应，但因为不具备红石导体的性质而无法被充能。

半连接激活: 发射器、投掷器与活塞能够在三种机械当上方的方块_能被_激活时也会被激活，即使这个“上方的方块”并不存在也没问题。
